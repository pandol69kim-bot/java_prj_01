## 작업 내용

### 서브태스크 2.1: BaseEntity + ExternalDataSync 엔티티
- `BaseEntity`: `@MappedSuperclass` + JPA Auditing (createdAt, updatedAt)
- `User`: username/email unique, Role(ADMIN/USER), enabled 필드
- `ExternalSystem`: systemCode unique, baseUrl, authType(NONE/API_KEY/BEARER/BASIC), active
- `ExternalDataSync`: ExternalSystem N:1, externalRefId, SyncStatus(PENDING/PROCESSING/COMPLETED/FAILED/SKIPPED), rawPayload/processedPayload, retryCount

### 서브태스크 2.2: SyncLog 엔티티
- `SyncLog`: ExternalSystem N:1, SyncLogStatus(RUNNING/COMPLETED/PARTIAL_FAILED/FAILED), TriggerType(SCHEDULED/MANUAL/API)
- totalCount/successCount/failCount 집계 필드
- `JpaConfig`: @EnableJpaAuditing, @EnableJpaRepositories

### 서브태스크 2.3: Repository 인터페이스
- `UserRepository`: findByUsername, findByEmail, existsBy
- `ExternalSystemRepository`: findBySystemCode, findAllByActiveTrue
- `ExternalDataSyncRepository`: 상태/retryCount 기반 조회, JPQL 집계 쿼리
- `SyncLogRepository`: 최근 로그, 기간별 조회, RUNNING 상태 조회

### 서브태스크 2.4: Flyway 마이그레이션
- `V1__create_users.sql`: users 테이블 + 초기 admin 계정
- `V2__create_external_systems.sql`: external_systems 테이블
- `V3__create_external_data_syncs.sql`: external_data_syncs 테이블 + FK
- `V4__create_sync_logs.sql`: sync_logs 테이블 + FK + CHECK 제약
- build.gradle: flyway-core, flyway-database-postgresql 의존성 추가
- application.yml: spring.flyway 설정 추가
