## 작업 내용

### 1. 문제 진단
- 백엔드 로그: 정상 기동 (`Tomcat started on port 8080`)
- 프론트엔드 로그: `ECONNREFUSED` 오류 - `/api/v1/auth/login` 프록시 실패
- 원인: `vite.config.ts`의 프록시 타겟이 `http://localhost:8080` → Docker 컨테이너 내부에서 `localhost`는 백엔드가 아닌 자기 자신을 가리킴

### 2. 백엔드 API 직접 테스트
- `curl -X POST http://localhost:8080/api/v1/auth/login` → 정상 응답
- DB users 테이블 확인: `admin` 계정 존재 (enabled=true)
- 비밀번호: `admin1234` (V6 마이그레이션에서 확인)

### 3. 수정 사항

#### vite.config.ts
- 변경 전: `target: 'http://localhost:8080'`
- 변경 후: `target: process.env.VITE_API_TARGET || 'http://backend:8080'`
- Docker 컨테이너 내부에서는 `backend`가 서비스 이름으로 해석됨

#### tsconfig.json
- `"types": ["vite/client"]` 추가
- `import.meta.env` TypeScript 오류 해결 (router/index.ts에서 사용)

### 4. 검증
- `curl http://localhost:5173/api/v1/auth/login` → JWT 토큰 정상 발급
- `docker compose build frontend` → 프로덕션 빌드 성공

## 생성/수정된 파일
- `frontend/vite.config.ts` — Docker 네트워크용 프록시 타겟 수정
- `frontend/tsconfig.json` — vite/client 타입 추가

## 특이사항 / 결정 사항
- **계정 정보**: username=`admin`, password=`admin1234`
- 로컬 개발(Docker 외부)에서는 `VITE_API_TARGET=http://localhost:8080` 환경변수 설정 필요
- Docker 개발 환경에서는 기본값 `http://backend:8080` 사용
