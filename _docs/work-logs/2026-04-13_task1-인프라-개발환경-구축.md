## 작업 내용

### 서브태스크 1.1: Docker Compose + PostgreSQL + Nginx
- `docker-compose.yml`: postgres, backend, frontend, nginx 서비스 정의
- `docker-compose.dev.yml`: 개발 환경 오버라이드
- `nginx/nginx.conf`: `/api/` → Spring Boot, `/` → Vue.js SPA 리버스 프록시

### 서브태스크 1.2: Spring Boot 3.x 초기화
- `backend/build.gradle`: web, JPA, security, webflux, JWT, Swagger, Lombok, Jacoco(80%) 의존성
- `backend/src/main/resources/application.yml`: dev/prod/test 프로파일 분리
- `Application.java`: @EnableScheduling 포함 진입점
- `ApiResponse<T>`: 공통 응답 envelope (success, data, error, meta)
- `BusinessException` + `GlobalExceptionHandler`: 공통 예외 처리
- `ApplicationTest.java`: H2 기반 컨텍스트 로드 테스트

### 서브태스크 1.3: Vue.js 3 초기화
- `frontend/package.json`: Vue3, Pinia, Axios, Vue Router, Vitest, Playwright
- `frontend/vite.config.ts`: `/api` 프록시 → localhost:8080
- `frontend/tsconfig.json`: strict 모드, `@/` 경로 별칭
- `src/main.ts`, `App.vue`: 진입점
- `src/router/index.ts`: login/dashboard 라우트, 인증 가드
- `src/stores/auth.ts`: Pinia 기반 토큰 관리
- `src/api/client.ts`: Axios 인스턴스 (JWT 자동 첨부, 401 처리)
- `src/pages/LoginPage.vue`, `DashboardPage.vue`: 기본 페이지

### 서브태스크 1.4: Docker 빌드 파일 + GitHub Actions
- `backend/Dockerfile`: 멀티스테이지 (builder + runtime), non-root 실행
- `frontend/Dockerfile`: 멀티스테이지 (builder + nginx)
- `frontend/nginx.conf`: SPA 라우팅, 정적 자산 캐싱
- `.github/workflows/ci.yml`: backend(JDK17+Gradle+PostgreSQL) / frontend(Node20+Vitest+ESLint) 병렬 CI
