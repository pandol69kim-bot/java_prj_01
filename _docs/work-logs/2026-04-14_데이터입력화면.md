## 작업 내용

### 공통 컴포넌트
#### BaseModal.vue (신규)
- Teleport to="body" 방식으로 렌더링 (z-index 문제 없음)
- 슬롯: default (body), footer
- Transition 애니메이션 (fade + slide)
- ESC/오버레이 클릭으로 닫기

### 외부 시스템 등록 (SystemsPage.vue)
- "시스템 등록" 버튼 → BaseModal 폼 오픈
- 필드: 시스템 코드, 이름, Base URL, 인증 방식, API 키, 설명
- 프론트 유효성 검증 (systemCode 패턴, URL 형식)
- 등록 성공 시 3초 후 사라지는 성공 메시지

### 프론트 버그 수정 (api/externalSystem.ts)
- ExternalSystem.systemName → name (백엔드 응답 필드명 불일치 수정)
- authType: 'BEARER' → 'BEARER_TOKEN' (enum 값 불일치 수정)
- SystemsPage.vue rows 매핑도 함께 수정

### 동기화 데이터 수동 입력 (SyncDataPage.vue)
- 시스템 선택 시 "데이터 추가" 버튼 표시
- BaseModal 폼: 외부 참조 ID, 처리 상태 선택, 원본 페이로드(선택)
- 추가 완료 후 목록 자동 새로고침

### 백엔드 수동 입력 API
#### ManualSyncRequest.java (신규)
- 필드: systemId, externalRefId, status, rawPayload, errorMessage
- Bean Validation 적용

#### ExternalDataSync.forManualEntry() (추가)
- 기존 factory method 패턴 확장
- COMPLETED 상태면 syncedAt = now() 자동 설정

#### SyncDataQueryService.createManual()
- ExternalSystemRepository 주입, 시스템 존재 여부 검증
- ExternalDataSync.forManualEntry()로 생성 후 저장

#### SyncDataController POST /manual
- ADMIN 권한 필요
- @Valid 검증
