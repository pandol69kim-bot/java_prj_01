### 7.3 Pinia 스토어 설정

- `src/stores/auth.ts` — 이전 세션(Task #1)에서 완료
- `src/stores/syncData.ts` — 외부 시스템 목록, 동기화 데이터, 대시보드 요약 상태 관리

`syncData` 스토어 주요 기능:
- `fetchSystems()` — 외부 시스템 페이징 조회
- `fetchActiveSystems()` — 활성 시스템 목록
- `fetchSyncItems(systemId, page)` — 시스템별 동기화 데이터 조회
- `fetchSummary()` — 대시보드 요약 조회
- `triggerSync(systemCode)` — 수동 동기화 트리거
- `retryFailed()` — 실패 건 재시도

## 특이사항 / 결정 사항

- **BasePagination**: 0-based 페이지 인덱스(Spring Pageable 기준)로 구현. 화면 표시는 +1 처리
- **StatusBadge**: 백엔드 enum 값을 직접 매핑. 알 수 없는 값은 원본 문자열을 회색 배지로 표시
- **AppHeader 사용자명**: localStorage JWT를 클라이언트에서 디코딩(Base64). 서버 호출 없이 표시
- **AppSidebar 접힘**: 아이콘만 표시되는 3.5rem 너비로 축소, CSS transition으로 애니메이션
- **syncData 스토어**: 에러 상태를 각 fetch별로 분리 관리하여 독립적인 UI 피드백 가능
- **nginx 버그 수정**: `proxy_pass` + `try_files` 충돌 버그 발견 및 수정 (→ `_docs/수정작업.md` 참고)
